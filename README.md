# Блог-платформа (Вариант 2)

REST API для управления статьями, комментариями и категориями с ролевой моделью и JWT-аутентификацией.

## Функциональность

- Регистрация с подтверждением email, вход, обновление токенов (access + refresh)
- Статьи: CRUD с фильтрацией по категориям, статусу, сортировкой (по дате/популярности) и пагинацией
- Комментарии к статьям с древовидной структурой (список, добавление, редактирование, удаление)
- Категории: CRUD (только модератор)
- Лайки статьям и комментариям (toggle-режим)
- Загрузка изображений к статьям (одиночная и множественная)
- Учёт просмотров статей (уникально на пользователя)
- Система ролей: читатель, автор, модератор
- Логирование всех операций изменения данных

## Установка и запуск

1. Клонируйте репозиторий:
   ```bash
   git clone <repo-url>
   cd blog-platform
   ```
2. Создайте виртуальное окружение
   > python -m venv .venv
3. Активируйте виртуальное окружение (.venv)
   > .venv\Scripts\activate
4. Установите зависимости:
   > python -m pip install -r requirements.txt
5. Настройте окружение – создайте файл **.env** в корне проекта на основе **.env.example**:  
   >JWT_SECRET = your_secret_key_change_me  
   JWT_REFRESH_SECRET = your_super_secret_key_change_me  
   JWT_EXPIRE_MINUTES = 10  
   JWT_REFRESH_EXPIRE_DAYS = 7  
   MAIL_USERNAME= your_email@gmail.com  
   MAIL_PASSWORD = your_app_password  
   MAIL_FROM = your_email@gmail.com  
   SMTP_SERVER = smtp.gmail.com  
   SMTP_PORT = 587  
   MEDIA_URL = /media  
   MEDIA_ROOT = media  
   ALLOWED_ORIGINS=localhost1,localhost2,... (опционально)

6. Создайте таблицы и наполните БД тестовыми данными:

   > python seed.py

7. Запустите сервер в режиме разработки:

   **Режим разработки (dev):**

   ```bash
   python -m fastapi dev main.py
   ```

   **Режим продакшена (prod):**

   ```bash
   python -m fastapi prod main.py
   ```

   - Сервер будет доступен по адресу **http://127.0.0.1:8000**.
   - Интерактивная документация (Swagger UI): **http://127.0.0.1:8000/docs**.

## Описание API

### Методология

REST API с JWT-аутентификацией (Bearer-токен).  
Все запросы на изменение данных требуют заголовка `Authorization: Bearer <access_token>`.  
Refresh-токен используется для получения нового access-токена.

---

### Формат передачи данных

- Эндпоинты регистрации, входа, создания/обновления сущностей принимают **form-data** (`application/x-www-form-urlencoded` или `multipart/form-data` для загрузки файлов).
- Ответы возвращаются в формате **JSON**.

**Параметры `GET /api/posts`:**

- `page`, `limit` — пагинация (по умолчанию 1 и 10)
- `category_ids` — список ID категорий (можно несколько)
- `status`:
  - `draft` — черновик
  - `published` — опубликовано
- `sort`:
  - `recent` — по дате публикации
  - `popular` — по количеству лайков

## Валидация ключевых эндпоинтов

Все входные данные проходят проверку через Pydantic и FastAPI. Основные случаи ошибок:

- **400 Bad Request**
  - Неверный формат списка ID категорий (`category_ids` должна быть строка с запятыми)
  - Попытка создать дублирующую категорию
  - Комментирование черновика (для не-автора)
  - Неверный старый пароль при смене пароля
- **401 Unauthorized** – отсутствует или недействителен токен
- **403 Forbidden** – недостаточно прав (например, читатель пытается создать статью)
- **404 Not Found** – сущность не существует или скрыта с учётом роли
- **422 Unprocessable Entity** – ошибки типов данных или отсутствие обязательных полей

## Роли пользователей и их права

### Читатель (reader)

- Видит только **опубликованные** статьи
- Может оставлять комментарии к опубликованным статьям
- Может ставить и убирать лайки статьям и комментариям
- Не может создавать, редактировать или удалять статьи и категории
- Может редактировать и удалять только свои комментарии

---

### Автор (author)

- Все права читателя
- Создание, редактирование и удаление **своих** статей в любом статусе
- Просмотр своих черновиков
- Комментирование своих черновиков
- Загрузка изображений к своим статьям

---

### Модератор (moderator)

- Полный доступ ко всем статьям: просмотр, редактирование и удаление, включая чужие черновики
- Удаление любых комментариев
- Управление категориями: создание, изменение и удаление
- Может создавать собственные статьи как автор
- Может комментировать любые статьи, включая черновики
